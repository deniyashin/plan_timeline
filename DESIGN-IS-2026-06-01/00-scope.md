# 00 — Scope Lock

## Что аудируем

- **Артефакт:** `plan_timeline.html` — единственная страница сайта
- **URL:** GitHub Pages (деплой из ветки `main`)
- **Поверхность:** весь интерфейс целиком, все 4 режима (Кратко / Подробно / Проекты / Редактирование)

## Первичный пользователь и задача

**Два уровня пользователей:**

1. **Учредители / топ-менеджмент** — читают, согласовывают, дают обратную связь по формулировкам, владельцам и срокам. Редактировать не умеют и не должны.
2. **Проектный офис** — ведут данные: редактируют тексты, управляют задачами, публикуют изменения через n8n webhook.

**Первичная задача пользователя уровня 1:** понять план изменений на 2026–2027, найти свои программы, оценить сроки и степень проработки — и либо согласовать, либо дать комментарий.

**Первичная задача пользователя уровня 2:** поддерживать данные актуальными: назначать исполнителей, обновлять статусы задач, добавлять проекты, публиковать.

## Ограничения

- **Стек:** Vanilla JS без бандлера, статический GitHub Pages
- **Нет TypeScript и тестов**
- **Язык:** русский (весь контент и UI)
- **Данные:** 5 изменений, 38 программ, 212 проектов, горизонт 2026–2027
- **Конкуренты/референсы:** Linear, Notion, корпоративные портальные дашборды

## Констатация

Аудит проводится против десяти принципов Дитера Рамса. Вердикт — один из трёх: NEW / REFINE / REDESIGN.
